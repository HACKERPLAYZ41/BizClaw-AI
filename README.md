# BizClaw AI - Multi-Tenant WhatsApp Business Platform

An advanced, SaaS-ready, multi-tenant WhatsApp AI Business Agent platform designed to run inside a **Pterodactyl Panel** (Node.js Egg) or general virtual server container. 

This platform allows an **Owner (Admin)** to sell WhatsApp AI chatbots to **Clients (Shop Owners / Businesses)**. Access is gate-locked via license keys: clients can only register with a valid key generated by the administrator. Each client manages their own isolated dashboard, pairs their own WhatsApp number, and configures their own Google Gemini or OpenAI credentials.

---

## 🔒 Security & Secrets Protection

To prevent API credential leakage, all sensitive global settings are stored in `config.yml` (which is excluded from Git via `.gitignore`).
Client-specific configuration objects (including client-submitted API keys) are stored securely inside the sandboxed database `database.json`, which is also ignored by Git.

---

## 🚀 Key Upgrades & Features

*   **Role Division (SaaS Ready):**
    *   **Owner (Admin):** Accesses a dedicated Admin Panel to generate unique license keys (specifying validity days and message quotas) and audit registered client stats.
    *   **Clients:** Manage their own isolated bot configurations, scan their WhatsApp device, check dynamic message counts, and read captured CRM leads.
*   **License Expiration & Limits:**
    *   Administrator sets active message quotas (e.g., `500 messages`) and duration (e.g., `30 days`) per license.
    *   If a client hits their message quota or passes their expiration date, the bot automatically pauses replies and requests the client to renew.
*   **Multi-Instance WhatsApp (Baileys):** Runs multiple parallel WhatsApp sockets concurrently inside a single Node.js process. Each client's authentication session is isolated under `auth_info/client_<username>/`.
*   **Sandboxed Sockets:** WebSocket rooms sandbox clients so they only receive updates and CRM leads related strictly to their own accounts.

## 🛠️ Step-by-Step SaaS Operations Guide

### 1. Setup Configuration
Boot the server once to automatically generate the `config.yml` configuration:
```bash
npm install
npm start
```
Edit the generated `config.yml` file to set your custom admin username and password:
```yaml
server:
  port: 3000
  admin_username: "utkarsh"                      # Custom Owner Username
  admin_password: "2402"                         # Custom Owner Password
  jwt_secret: "some_secure_signing_secret_here"  # Secret used to sign user session tokens

ai:
  global_gemini_api_key: ""                      # (Optional) Fallback key if clients do not have their own
  global_openai_api_key: ""                      # (Optional) Fallback key if clients do not have their own
```

### 2. Generate Licenses (As Owner)
1. Open your browser and go to `http://localhost:3000`.
2. Enter your custom `admin_username` and `admin_password` (defined in your `config.yml`).
3. Under the **Licenses** tab, select the number of days (e.g., `30 Days`) and message limit (e.g., `500 Messages`), then click **Create License Key**.
4. Copy the generated key (e.g., `ABCD-EFGH-IJKL`).

### 3. Client Registration
1. Send the generated license key to your client.
2. The client visits your dashboard, toggles the selector to **Register**, and creates their custom username, password, and inputs their license key.
3. Upon registration, their isolated bot environment is created.

### 4. Client Operations
1. The client logs in with their newly created username.
2. Under the **WhatsApp Status** page, they scan the pairing QR code to link their WhatsApp account.
3. Under **AI Settings**, they configure their business FAQs, select pre-written AI templates (Retail, Restaurant, or Salon/Service), and (optionally) input their own Google Gemini or OpenAI API keys.

## 📁 Multi-Tenant Directory Structure

*   [`database.json`](file:///database.json) - Stores all user accounts, active licenses registry, CRM leads, and message histories.
*   [`auth_info/client_<username>/`](file:///auth_info/) - Sandboxed folders containing the WhatsApp session keys for each client.
*   [`whatsapp-handler.js`](file:///whatsapp-handler.js) - Manages the pool of active WhatsApp sockets and quota updates.
*   [`server.js`](file:///server.js) - Role authorization check and REST API endpoints.
*   [`public/`](file:///public/) - Glassmorphic SPA frontend dashboard.
